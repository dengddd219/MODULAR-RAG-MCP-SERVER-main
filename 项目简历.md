# 模块化RAG MCP服务器项目简历

## 项目概述

**项目名称**：Modular RAG MCP Server（模块化检索增强生成MCP服务器）  
**项目类型**：企业级RAG（检索增强生成）系统  
**开发周期**：持续迭代中  
**技术栈**：Python 3.10+, ChromaDB, Streamlit, MCP Protocol, LLM APIs  
**项目定位**：可插拔、可观测的模块化RAG服务框架，通过MCP协议对外暴露工具接口

---

## 核心亮点

### 🎯 技术深度
- **混合检索引擎**：实现Dense（语义向量）+ Sparse（BM25关键词）双重检索，通过RRF（Reciprocal Rank Fusion）算法融合，显著提升查全率和查准率
- **智能重排序机制**：支持Cross-Encoder和LLM Rerank两种精排策略，采用"粗排召回→精排过滤"两段式架构，在保证响应速度的同时提升Top-K结果精准度
- **多模态图像处理**：基于LLM视觉能力实现Image-to-Text转换，将图像描述嵌入文档块，实现"搜文字出图"的语义检索能力

### 🏗️ 架构设计
- **全链路可插拔架构**：采用抽象接口设计，支持LLM、Embedding、Rerank、Loader、Splitter等核心组件的零代码切换
- **多提供商支持**：无缝支持Azure OpenAI、OpenAI、Ollama、DeepSeek等多种LLM和Embedding服务，便于成本优化和A/B测试
- **MCP生态集成**：遵循Model Context Protocol标准，可直接对接GitHub Copilot、Claude Desktop等AI助手，实现零前端开发的智能问答

### 📊 工程化能力
- **可观测性系统**：实现全链路追踪（TraceContext），支持摄取和查询过程的阶段耗时分析、结果对比可视化
- **可视化Dashboard**：基于Streamlit构建六页管理平台（总览/数据浏览/摄取管理/追踪分析/评估面板），提供完整的系统监控能力
- **评估体系**：集成Ragas评估框架和自定义评估器，支持多维度指标（相关性、准确性、完整性等）的自动化评测

## 技术实现细节

### 1. 数据摄取流水线（Ingestion Pipeline）

**职责**：构建统一、可配置且可观测的文档处理流水线

**核心实现**：
- **文件完整性校验**：基于SHA256哈希实现去重和增量更新机制
- **智能文档解析**：支持PDF、Markdown等多种格式，提取文本、图片和元数据
- **语义感知分块**：采用递归切分策略，保留完整语义边界，支持可配置的chunk_size和overlap
- **多模态增强**：
  - Chunk精炼：使用LLM优化分块质量，提升语义完整性
  - 元数据增强：自动提取文档标题、页码、章节等结构化信息
  - 图像描述生成：利用Vision LLM为文档中的图片生成详细文字描述
- **批量向量化**：实现Dense Embedding和Sparse BM25双重编码，支持批量处理和异步优化
- **多存储后端**：统一存储到ChromaDB向量库、BM25索引和图片存储系统

**技术亮点**：
- 采用Pipeline模式，支持阶段级别的错误处理和优雅降级
- 实现TraceContext追踪，记录每个阶段的耗时和结果
- 支持配置驱动的组件替换，便于A/B测试和算法迭代

### 2. 混合检索引擎（Hybrid Search Engine）

**职责**：实现高精度、高召回率的混合检索系统

**核心实现**：
- **查询预处理**：QueryProcessor提取关键词和元数据过滤器
- **并行检索**：
  - Dense Retriever：基于向量相似度的语义检索，使用ChromaDB的向量搜索能力
  - Sparse Retriever：基于BM25的关键词检索，支持精确匹配和专有名词查找
  - 采用ThreadPoolExecutor实现并行检索，提升响应速度
- **结果融合**：实现RRF（Reciprocal Rank Fusion）算法，动态融合两种检索结果
- **优雅降级**：当一种检索方式失败时，自动降级到另一种方式，保证系统可用性
- **后置过滤**：支持基于元数据的后置过滤，满足细粒度检索需求

**技术亮点**：
- 实现可配置的top_k参数（dense_top_k、sparse_top_k、fusion_top_k）
- 支持元数据过滤（collection、doc_type、tags等）
- 完整的TraceContext集成，记录检索过程的详细指标

### 3. 重排序系统（Reranker）

**职责**：对混合检索结果进行深度语义排序

**核心实现**：
- **多策略支持**：
  - Cross-Encoder Reranker：使用专用重排模型（如bge-reranker）进行query-document对打分
  - LLM Reranker：利用大语言模型的深度理解能力进行重排序
- **可插拔设计**：通过抽象接口支持不同reranker的切换
- **性能优化**：仅在top_k候选集上进行重排，平衡精度和效率

### 4. MCP服务器（MCP Server）

**职责**：通过Model Context Protocol标准协议对外提供服务

**核心实现**：
- **标准协议实现**：基于官方MCP SDK实现stdio传输协议
- **工具接口**：
  - `query_knowledge_hub`：知识库查询工具，支持自然语言问答
  - `list_collections`：列出所有文档集合
  - `get_document_summary`：获取文档摘要信息
- **日志隔离**：确保stdout仅包含JSON-RPC协议消息，所有日志输出到stderr
- **异步处理**：基于asyncio实现高并发请求处理

**技术亮点**：
- 完全符合MCP标准，可与GitHub Copilot、Claude Desktop等客户端无缝集成
- 支持环境变量和配置文件两种配置方式
- 实现完整的错误处理和日志记录

### 5. 可视化Dashboard

**职责**：提供系统监控、数据管理和性能分析的可视化界面

**核心实现**：
- **六页管理平台**：
  - Overview：系统总览，展示组件配置和数据资产统计
  - Data Browser：数据浏览器，支持按collection查看文档、Chunk内容和图片预览
  - Ingestion Manager：摄取管理，支持文件上传、触发摄取、进度监控和文档删除
  - Ingestion Traces：摄取追踪，展示摄取历史、阶段耗时瀑布图和trace详情
  - Query Traces：查询追踪，对比Dense/Sparse检索结果和Rerank前后效果
  - Evaluation Panel：评估面板，运行评估任务、展示指标和历史趋势
- **实时数据更新**：基于Streamlit的响应式设计，支持实时数据刷新
- **交互式可视化**：使用图表和表格展示检索结果对比、性能指标等

### 6. 评估体系（Evaluation Framework）

**职责**：提供多维度、自动化的RAG系统评估能力

**核心实现**：
- **Ragas集成**：使用Ragas框架评估相关性、准确性、完整性等指标
- **自定义评估器**：支持业务特定的评估逻辑
- **Golden Test Set**：支持基于标准测试集的回归测试
- **历史趋势分析**：记录评估历史，支持性能趋势可视化

## 核心算法与设计模式

### 1. 混合检索算法（Hybrid Search）
- **RRF融合**：实现Reciprocal Rank Fusion算法，动态融合Dense和Sparse检索结果
- **并行检索**：使用ThreadPoolExecutor实现两种检索方式的并行执行
- **优雅降级**：当一种检索失败时自动切换到另一种，保证系统可用性

### 2. 设计模式应用
- **工厂模式**：LLMFactory、EmbeddingFactory、RerankerFactory等，支持动态创建不同提供商实例
- **策略模式**：Splitter、Loader、Transform等组件采用策略模式，支持运行时切换
- **管道模式**：IngestionPipeline采用管道模式，支持阶段级别的错误处理和追踪
- **观察者模式**：TraceContext实现全链路追踪，记录每个阶段的执行情况

### 3. 性能优化
- **批量处理**：BatchProcessor实现向量编码的批量处理，提升吞吐量
- **增量更新**：基于SHA256的文件完整性校验，支持增量摄取
- **异步处理**：MCP服务器采用asyncio实现异步请求处理

## 技术难点与解决方案

### 1. 混合检索结果融合
**难点**：如何平衡Dense和Sparse两种检索方式的结果，确保既保留语义相似性又保留关键词精确匹配  
**解决方案**：实现RRF（Reciprocal Rank Fusion）算法，通过动态权重融合两种检索结果，并通过大量实验调优RRF参数

### 2. 多模态图像检索
**难点**：如何在纯文本RAG系统中实现图像检索能力  
**解决方案**：采用Image-to-Text策略，利用Vision LLM生成图像描述，将描述嵌入文档块，复用现有文本检索链路

### 3. 可插拔架构设计
**难点**：如何设计抽象接口，支持不同提供商的零代码切换  
**解决方案**：采用工厂模式和策略模式，定义统一的抽象接口（BaseLLM、BaseEmbedding等），各提供商实现接口，通过配置文件动态选择

### 4. 全链路追踪
**难点**：如何在复杂的Pipeline中追踪每个阶段的执行情况和性能指标  
**解决方案**：实现TraceContext上下文管理器，在每个阶段记录耗时、输入输出和错误信息，支持JSON序列化和可视化展示
